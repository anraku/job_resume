# 自己PR
　新卒で入社したSI企業では主にサーバサイドエンジニア、インフラエンジニアを経験してきました。学生時代でのプログラミングの経験や学習したITの知識を生かして退職までに8つほどのプロジェクトに参画し、要件定義から実装、テスト、プロジェクトによっては保守・運用作業までをワンストップで担当しました。使用した技術は主にPHP、Java（WebアプリとAndroidアプリで使用）で、他にもPython、C#、Swiftを実務で使用した経験があります。

　2社目の会社ではインフラエンジニア、サーバサイドエンジニアとしてRailsやGo言語を使ったアプリケーション開発や運用保守業務に従事していました。定常的な業務としてFluentd、Mackerelなどをプロダクト運用に導入しているチームでプロダクトの運用・改善に取り組んできました。また、Go言語を使った社外向けWebAPIの新規開発や、Ruby on Railsを使った社内システムの新規開発も担当しました。それ以外にも開発環境の構築、整備を行ったり、デプロイの自動化、プログラムのドキュメント作成など必要と思われた作業を自ら考え、実践していくことを好のんで行動してきました。

　現在までに2年ほどの運用保守業務の業務経験があり、システム全体を俯瞰して見る能力、障害発生時の問題解決能力には自信があります。運用保守以外でも様々なプロジェクトを通してJava, C#, PHP, Python, Ruby, Golangなどのプログラミング言語やLinuxサーバの保守業務を通してBashスクリプトの作成やサーバ監視についての知見を身に着けてきており、学びながらプロジェクトに貢献していくことにやりがいを感じます。 また、開発チームの生産性向上を図ってTDDやペアログラミングなどの開発手法を実験的に取り入れてみるなど、良いと思ったことはチーム内で提案して実践することが好きです。

## 某金融系企業の業務システムの開発 2016年
某金融系企業の業務システムの一部を改修

### 担当業務
要件定義、基本設計、詳細設計、実装、ソースレビュー、テスト、本番リリース
某金融系企業の業務システムの改修案件にて要件定義からリリースまで一貫して担当

### 習得スキル
- 顧客との要件定義
- 設計書作成
- Javaアプリケーションの追加改修
- JUnitを使った単体テストの実装
- テスト項目書の作成
- Selenium IDEを使ったブラウザテスト
- JMeterを使った負荷テストの環境構築と実施方法
